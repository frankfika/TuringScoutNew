# Leaderboard-First Strategy

## 1. Strategic Shift

AICookieDao should start as a leaderboard and discovery media product, not as a heavy task marketplace.

The first product experience should be:

> Come here to see which AI projects, agents, open-source repos, credit campaigns, and reward opportunities are trending today.

This creates free traffic, SEO pages, social sharing, project-side self-promotion, and repeat user habits before requiring users or projects to register.

## 2. Why Leaderboard First

### Lower User Friction

Users can immediately browse rankings without creating an account. This is ideal for a “薅羊毛” audience because they want fast opportunity scanning.

### Better Organic Distribution

Rankings naturally generate sharing:

- Projects share “we are ranked #3 this week”
- Users share “top AI free credits this week”
- Creators use rankings as content material
- Communities repost useful lists

### Easier SEO

Leaderboard pages target search intent better than generic campaign pages:

- best AI free credits
- AI agent leaderboard
- AI airdrops
- open-source AI projects trending
- GitHub AI projects fastest growing
- AI bounty list
- AI tools with free credits
- MCP server leaderboard

### Easier Project Acquisition

Projects care about being ranked, featured, compared, and discovered. A leaderboard gives them a reason to submit information even before paid campaigns exist.

### Commercialization Comes Later

Once rankings have traffic, monetization becomes easier:

- Featured placement
- Sponsored launch slot
- Verified project badge
- Analytics report
- Campaign boost
- Newsletter sponsorship

## 4. Main Leaderboards

### 1. Today’s AI Opportunities

Purpose: daily homepage feed.

Ranks AI opportunities by freshness, reward value, project credibility, and ease of participation.

Good for:

- Daily return traffic
- Telegram/newsletter updates
- Social posts

### 2. AI Free Credits Leaderboard

Ranks AI projects offering API credits, compute credits, model credits, SaaS trials, or free plans.

Audience:

- Developers
- Indie hackers
- students
- AI tool users

SEO keywords:

- AI free credits
- free AI API credits
- free LLM credits
- AI tools free trial

### 3. AI Points / Possible Airdrop Leaderboard

Ranks AI projects with points, quests, usage rewards, referral systems, or possible future token incentives.

Audience:

- Airdrop hunters
- Web3 x AI users
- early adopters

Important: label uncertainty clearly. Do not promise airdrops.

### 4. Open-Source AI Momentum Leaderboard

Ranks AI open-source projects by star velocity, fork velocity, contributors, release cadence, issue activity, social attention, and task availability.

Audience:

- Developers
- investors
- DevRel teams
- technical creators

This is the best bridge between ProductHunt and GitHub Trending.

### 5. AI Agent Leaderboard

Ranks agents by usage opportunity, demo quality, task usefulness, integrations, community interest, and reward campaigns.

Categories:

- Coding agents
- Browser agents
- Research agents
- Sales/marketing agents
- Data agents
- Personal productivity agents
- MCP/tool agents

### 6. AI Bounty Leaderboard

Ranks cash bounties, grants, hackathons, creator tasks, integration challenges, and technical contribution opportunities.

Audience:

- Developers
- creators
- students
- freelancers

### 7. MCP / Agent Tooling Leaderboard

Ranks MCP servers, agent frameworks, memory tools, RAG tools, eval tools, workflow automation tools, and LLMOps tools.

This can become a strong technical SEO asset.

## 5. Ranking Logic

Start with transparent but simple scoring. Do not over-engineer.

```
Rank Score = Opportunity Value + Momentum + Credibility + Ease + Community Interest - Risk
```

### Opportunity Value

- Cash reward
- Free credit value
- Grant size
- Possible future upside
- Early access scarcity
- Career/reputation value

### Momentum

- GitHub star velocity
- Social mentions
- Product launch recency
- Discord/community growth if visible
- Newsletter/social engagement
- User clicks on AICookieDao

### Credibility

- Official website
- Real team or company
- GitHub quality
- Funding/partners if public
- Product availability
- Clear reward terms

### Ease

- No login required
- Quick signup
- Clear task steps
- Low technical complexity
- Low privacy risk

### Community Interest

- Outbound clicks
- Saves
- User submissions
- Reports from hunters
- Creator mentions

### Risk

- Unclear reward
- Possible scam
- Fake project
- Spam-like requirements
- Overly invasive permissions
- No official source

## 6. Traffic Loops

### Project Sharing Loop

1. Project appears on leaderboard.
2. Project gets a badge: “Trending on AICookieDao”.
3. Project shares ranking on X/Discord/website.
4. New users visit leaderboard.
5. More projects submit themselves.

### Hunter Return Loop

1. User checks daily/weekly leaderboard.
2. User clicks interesting opportunities.
3. User subscribes to alerts.
4. User submits new opportunities they find.
5. Platform gets fresher content.

### Creator Content Loop

1. AICookieDao publishes weekly ranking.
2. Creators use it to make threads/videos/posts.
3. AICookieDao reposts good content.
4. Creators get audience; platform gets backlinks and traffic.

### SEO Loop

1. Each category leaderboard becomes an evergreen page.
2. Each project gets an indexable page.
3. Each opportunity gets a task page.
4. Weekly archives create long-tail pages.
5. Search traffic compounds.

## 8. Low-Registration Product Flow

### Anonymous User

Can:

- Browse all leaderboards
- Open project pages
- Click official task links
- Share rankings
- Subscribe to newsletter/Telegram
- Submit a project/opportunity with email

Cannot:

- Save tasks
- Mark completed
- Claim platform-intermediated rewards
- Build public reputation profile

### Optional Logged-In User

Can:

- Save opportunities
- Mark completed
- Get reminders
- Submit proof
- Build profile
- Join hunter leaderboard

Registration appears only after repeated value is demonstrated.

## 9. First 100 Listings Strategy

Curate manually across these buckets:

- 20 AI free credit opportunities
- 20 open-source AI projects with momentum
- 15 AI agent trials
- 15 points/possible airdrop programs
- 15 developer bounties/grants/hackathons
- 15 MCP/RAG/agent tooling projects

Each listing should have:

- Clear title
- Official link
- Reward/opportunity type
- Difficulty
- Credibility note
- Risk note
- Last updated date

## 10. Monetization After Traffic

Do not lead with monetization before ranking credibility exists.

Monetization sequence:

1. Free curated listings
2. Featured project slots
3. Sponsored but labeled launch placement
4. Verified project badge
5. Weekly newsletter sponsorship
6. Managed campaign package
7. Project analytics report
8. API/data products

## 11. Editorial Rules

To keep the leaderboard trusted:

- Paid placement must be labeled
- Organic ranking cannot be bought
- Ranking cannot be purely pay-to-rank
- AI-generated ranking explanations should be visible
- Unverified rewards must be labeled
- Risky projects can be listed with warnings or rejected
- GitHub stars must not be sold or guaranteed
- Spam tasks should be rejected
- Official links should be prioritized

## 12. MVP Implication

The first version should prioritize automation, AI-native scoring, fairness, and:

- Public leaderboard pages
- Filterable opportunity feed
- SEO-friendly project pages
- Submission form
- Shareable ranking badges/cards
- Newsletter/Telegram capture
- Basic click tracking

Delay:

- Complex reward claiming
- Heavy user profiles
- Automated proof verification
- Token/DAO mechanics
- Full campaign marketplace
