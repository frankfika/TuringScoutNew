# 早期路线图讨论

> 归档说明：本文档是产品路线图的早期讨论版本，已被 01-product-roadmap.md 替代。

## 早期版本设想

### 初始阶段划分（v1）

```
Phase 1: 数据看板（4周）
  - GitHub Trending 聚合
  - 基础项目展示
  - 无用户系统

Phase 2: 社区层（4周）
  - Scout 提交
  - 积分系统
  - 排行榜

Phase 3: 代币层（4周）
  - 代币部署
  - 质押机制
  - 治理
```

## 为什么调整

### 问题 1：代币太早
原来的计划是 Phase 3 就上代币，但在可行性评估后发现：
- V1 的用户量不足以支撑代币经济
- 合规风险需要在有用户基础后再评估
- 代币会分散核心产品开发精力

### 问题 2：阶段太紧凑
3 个 Phase 各 4 周过于乐观，实际发现：
- 数据质量打磨需要更多时间
- Scout 系统需要运营陪跑
- 后台管理比预期复杂

### 问题 3：缺少验证节点
原计划缺少明确的"通过/不通过"决策点，新版本在每个里程碑设置了通过条件。

## 调整后的版本划分

调整为 V0-V5 的渐进式路线图（详见 01-product-roadmap.md），核心变化：
1. 代币推迟到 V5（最早）
2. 每个版本有明确的验证标准
3. 增加了 V1.5 运维阶段
4. 在 V2 之前先验证流量和数据价值

## 关键决策记录

| 决策 | 原方案 | 调整后 | 原因 |
|------|--------|--------|------|
| 代币时间 | V1 内 | V5 | 合规 + 用户基础不足 |
| 用户系统 | V1 必做 | V1 不做 | 降低门槛 |
| 创作者内容 | V1 必做 | V2 | 先验证数据价值 |
| 多数据源 | V1 选做 | V1 必做 | 差异化核心 |
| Scout 积分 | V2 | V1 | 数据质量保障 |
