# Automation, AI-Native, Fairness Principles

## 1. Core Philosophy

AICookieDao should be built around three non-negotiable principles:

1. **Automation**: the platform should discover, structure, score, rank, publish, and report with minimal manual work.
2. **AI-Native**: AI should not be a feature decoration; it should be the operating system of the product.
3. **Fairness**: rankings should not become pure pay-to-rank, fake-star delivery, or insider traffic allocation.

The goal is to build a lightweight, self-updating AI opportunity index where projects compete on real momentum, reward quality, usefulness, and community interest.

## 2. Automation

### What Should Be Automated

- Discover new AI projects and opportunities
- Extract project/reward/task information from official sources
- Deduplicate projects and campaigns
- Categorize opportunities
- Score opportunities
- Generate listing summaries
- Generate risk notes
- Generate ranking explanations
- Generate SEO pages
- Generate weekly digests
- Generate project outreach drafts
- Generate basic performance reports

### What Should Stay Human-Controlled

- Final review for suspicious opportunities
- Sponsored placement approval
- Scam/risk escalation
- Editorial taste for homepage highlights
- Project-side sales conversations
- Major policy decisions

### Automation Goal

Daily operation should not require a large ops team. A single operator should be able to review and approve the day’s AI-generated candidates in less than 30 minutes.

## 3. AI-Native

### AI As The Product Engine

AI should power the core user experience:

- “Why this project is trending”
- “Why this opportunity is ranked”
- “Best for developers / hunters / creators / founders”
- “Reward clarity”
- “Risk warning”
- “Similar opportunities”
- “Official-source summary”
- “What changed since last week”

### AI As The Editorial Team

Instead of manually writing every listing, AI drafts:

- Project intro
- Opportunity card
- Task steps
- SEO metadata
- Social copy
- Newsletter copy
- Weekly trend analysis

Human review improves quality but should not be the bottleneck.

### AI As The Analyst

AI should compare projects across categories:

- Which open-source AI repos are gaining momentum?
- Which AI agents have real usage opportunities?
- Which campaigns have high reward but high risk?
- Which free credit offers are actually valuable?
- Which projects look under-discovered?

## 4. Fairness

Fairness is critical because leaderboards can easily become untrusted.

### Ranking Fairness

The default ranking should combine:

- Public momentum
- Reward/opportunity value
- Project credibility
- User interest on AICookieDao
- Ease of participation
- Risk score
- Freshness

Paid placement can exist, but it must be labeled and separated from organic ranking.

### Project Fairness

Small projects should have a chance to rank if they offer high-value opportunities or show strong momentum.

Avoid a system where only funded companies can dominate rankings.

Recommended mechanics:

- Organic leaderboard cannot be directly bought
- Sponsored slots are clearly labeled
- New projects get a freshness boost
- Category-specific rankings prevent large projects from dominating everything
- Rising projects get highlighted separately from absolute leaders

### User Fairness

Users should not be tricked into low-quality or risky tasks.

Required labels:

- Verified
- Unverified
- Sponsored
- Official Link
- Reward Not Guaranteed
- High Risk
- Ending Soon
- No Login
- Beginner Friendly
- Builder Task

### Creator/Builder Fairness

If future rewards or contribution profiles are added:

- Reward real work, not bot activity
- Do not over-reward follower count only
- Value technical contribution, feedback, integration, and useful content
- Explain why a contribution is accepted or rejected
- Provide appeal/report mechanisms

### Anti-Pay-To-Win Rule

AICookieDao can sell visibility, but not trust.

Allowed:

- Sponsored listing
- Featured launch slot
- Newsletter sponsorship
- Project analytics report
- Campaign design service

Not allowed:

- Guaranteed GitHub stars
- Guaranteed ranking position inside organic leaderboard
- Fake reviews
- Bot traffic
- Undisclosed sponsored posts
- Spam tasks

## 5. Transparent Scorecard

Each opportunity page should show a simple scorecard:

- Opportunity Value
- Project Credibility
- Ease
- Momentum
- Community Interest
- Risk
- Last Checked

Users do not need to see the full algorithm, but they should understand why something is ranked.

Example:

```
Ranked because:
- Offers meaningful free API credits
- Official campaign page is available
- GitHub repo is growing this week
- Task can be completed in under 5 minutes
- Reward terms are clear
```

## 6. Organic vs Sponsored Display

### Organic Ranking

Determined by algorithmic score and editorial safety review.

### Sponsored Slots

Can appear in clearly labeled areas:

- Sponsored Launch
- Featured Opportunity
- Partner Campaign

Sponsored slots should not secretly change organic rankings.

### Verified Badge

Verified means:

- Official source checked
- Project ownership or official links confirmed
- Reward details are reasonably clear

Verified does not mean:

- Reward is guaranteed
- Airdrop is guaranteed
- Project is risk-free

## 7. Anti-Manipulation

Signals to monitor:

- Sudden unnatural click spikes
- Repeated clicks from suspicious sources
- Project self-submission spam
- Fake social proof
- GitHub star anomalies
- Duplicate project submissions
- Misleading reward claims

Responses:

- Reduce score
- Add risk label
- Require human review
- Temporarily hide listing
- Ban malicious submissions

## 8. Product Copy Principles

Use fair and careful language.

Good:

- “Possible airdrop”
- “Reward not guaranteed”
- “Official task link”
- “AI-generated summary from public sources”
- “Ranked by momentum, value, and credibility”
- “Sponsored placement”

Avoid:

- “Guaranteed airdrop”
- “Buy GitHub stars”
- “Guaranteed ranking”
- “Risk-free”
- “Official partner” unless confirmed

## 9. Fairness As A Growth Strategy

Fairness is not only ethics; it is growth.

Users return when rankings feel useful and not bought.
Projects submit when they believe they have a fair chance.
Creators cite the platform when the methodology feels credible.
Investors and builders trust the data when manipulation is controlled.

Long-term moat comes from trusted AI-ranked opportunity data, not from short-term paid placements.

## 10. Implementation Priority

V1 should include:

- AI-generated score rationale
- Sponsored labels
- Verified/unverified labels
- Risk labels
- Last checked timestamp
- Report issue button
- Separate organic ranking and sponsored modules
- Category rankings for smaller projects

V2 can add:

- Public methodology page
- User voting with anti-abuse controls
- Contributor reputation
- Project claim/verification
- Appeal flow
- More advanced anomaly detection
