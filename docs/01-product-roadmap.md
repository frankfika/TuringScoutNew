# 01 — 产品路线图

## 概述

TuringScout 采取渐进式发展策略：先验证数据价值，再构建社区网络，最后引入代币经济。每个版本在上一个版本验证成功后启动。

## V0 — 战略和文档沉淀（已完成）

**目标**：完成产品定位、市场调研、技术可行性评估

**交付物**：
- 00-10 完整文档体系
- 竞品分析（CookieDAO、GitHub Trending、Hugging Face 等）
- 技术可行性验证（GitHub API、HN API 数据可获取性确认）

**不做**：
- 任何代码开发
- 任何部署操作

**成功标准**：文档体系完整，逻辑自洽，可作为开发依据

---

## V1 — 全自动 AI 项目机会榜

**目标**：上线一个完全由 AI 自动驱动的机会榜单，用户只读浏览。AI 自动抓数据、AI 自动写点评、AI 自动打分。

**核心功能**：
- 数据自动抓取（GitHub Trending、HN、Hugging Face）
- AI 信息抽取 + AI 深度点评生成（亮点/适合谁/同类对比/打分解读）
- 评分排序（多维评分，每维度有 AI 解读）
- 分类筛选（免费试用、开源项目、积分奖励、奖金竞赛）
- 项目详情页（GitHub 指标、关联机会）
- 后台管理（AI抓取审核、点评修改、数据源管理、评分调整）

**边界**：
- 纯只读，无用户系统
- 不开放用户提交（推迟到 V1.5）
- 不做社区/UGC
- 不做个性化推荐
- 不做付费功能

**不做**：
- L2 链部署
- 创作者内容系统
- Campaign 系统
- API 对外开放
- Scout 提交系统
- 代币合约

**成功标准**：
- 100+ AI 项目收录，每条有 AI 自动点评
- 数据更新延迟 < 1 小时
- 三方数据源正常运行

---

## V1.5 — 自动化运维和分发

**目标**：内容自动生成和分发，降低人工运营成本

**核心功能**：
- 自动化周报/日报生成
- SEO 优化（分类页、项目页、标签页）
- Newsletter 邮件分发
- 社交媒体自动发布（Twitter Bot、Telegram Channel）
- 监控和告警（数据源异常、评分异常）
- **Scout 提交系统引入**（V1.5 首次开放用户提交）

**不做**：
- 代币合约
- 社区功能

**成功标准**：
- DAU 1000+
- 周报订阅 500+
- SEO 流量占比 > 40%

---

## V2 — 创作者赋能和归因

**目标**：引入内容创作者，构建内容飞轮

**核心功能**：
- 创作者内容发布（解读、教程、基准测试）
- 内容-项目归因链接
- 创作者排行榜（Top Voices）
- 分享卡片（社交媒体优化）
- 内容审核系统

**不做**：
- 代币合约
- 付费墙
- Campaign

**成功标准**：
- 50+ 活跃创作者
- 外部引用增长 3x
- 用户停留时长增长 2x

---

## V3 — Campaign 和激励网络

**目标**：通过 Campaign 机制激活生态

**核心功能**：
- 项目方 Campaign（Launch Boost、Bounty）
- Scout 激励计划
- 创作者激励计划
- 积分系统升级（为代币做准备）

**不做**：
- 代币上线交易所
- DeFi 集成

**成功标准**：
- 10+ 项目方 Campaign
- Scout 贡献者 200+
- 积分系统运行稳定

---

## V4 — 数据/API 情报产品

**目标**：通过 API 和数据产品变现

**核心功能**：
- REST API（项目数据、趋势数据、评分数据）
- Webhook 通知
- 企业定制报告
- 数据导出（CSV、JSON）
- API 文档和 SDK

**成功标准**：
- 20+ API 付费客户
- 企业报告 5+ 份
- API 可用性 99.9%

---

## V5 — 代币/积分经济层

**目标**：上线代币经济，完成三层飞轮闭环

**核心功能**：
- 代币合约部署（ERC-20）
- 质押解锁高级功能
- API 调用燃烧代币
- 治理投票
- 生态空投
- 流动性池

**前提条件**：
- V1-V4 全部验证通过
- 用户基础 5万+ DAU
- 合规审查通过
- 社区治理框架成熟

**成功标准**：
- 代币持有地址 10,000+
- 质押率 > 30%
- 治理参与率 > 15%

---

## 版本依赖关系

```
V0 ──→ V1 ──→ V1.5 ──→ V2 ──→ V3 ──→ V4 ──→ V5
         │                │        │
         └── 数据飞轮 ────┴── 内容飞轮 ──┴── 代币飞轮
```

## 关键决策点

| 里程碑 | 决策点 | 通过条件 |
|--------|--------|---------|
| V1 完成后 | 是否继续 V1.5 | DAU > 500，回访率 > 20% |
| V1.5 完成后 | 是否启动 V2 | DAU > 1000，Scout 提交质量可控 |
| V2 完成后 | 是否启动 V3 | 创作者 > 50，外部引用 > 100 |
| V3 完成后 | 是否启动 V4 | 付费意愿验证通过（用户调研） |
| V4 完成后 | 是否启动 V5 | 监管合规确认，用户基础达标 |
